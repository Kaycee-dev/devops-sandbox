## TEACH-BACKS

### TEACH-BACK: Reuse existing env on duplicate create name

**Context.** Constitution §7.1 allows either same-env reuse or a clear conflict, but AGENTS.md highlights `create_env.sh foo` twice should give the same env back rather than two. Reuse also makes hurried demos less fragile.

**Alternatives considered**
1. **Reject duplicate names** — clearer API semantics, but CLI users can accidentally break the demo by rerunning `make create`.
2. **Return the existing active env** — idempotent for operators and matches the "same env back" guidance.

**Chosen** — **return the existing active env**, because it satisfies §7.1 without creating duplicate containers, networks, or Nginx routes.

**Failure modes**
- A stale state file can make create return an env whose container is gone; the Sprint 3 daemon reconciler must mark that state `error`.
- API callers may expect 409 conflict; the API will document reuse behavior in the response path instead of treating it as an error.

**Reversal cost.** Medium after the Postman collection starts relying on reuse semantics.

**Citations.**
- `governance/01_CONSTITUTION.md` §7.1
- `AGENTS.md` §4, Idempotent operations

### TEACH-BACK: Path-based Nginx snippets for env routes

**Context.** The API contract's state schema uses `http://localhost:18080/env-.../` URLs, while `06_DEFINITION_OF_DONE.md` says each per-env conf should be a single `server { }` block. Multiple duplicate `server_name _` blocks cannot reliably route different path prefixes.

**Alternatives considered**
1. **One server block per env** — follows the DoD wording but conflicts with path-based URLs and risks ignored duplicate server names.
2. **One generated `location` snippet per env inside the main server** — matches the API URL shape and lets Nginx route many envs through one host port.

**Chosen** — **path-based snippets**, because the public URL contract and single-front-door behavior matter more than the lower-level DoD wording. CRISIS mode says to decide and document rather than stop for an interactive clarification.

**Failure modes**
- Acceptance reviewers may expect `server { }` blocks from the DoD checklist; the journal and README need to explain the URL-contract trade-off.
- Bad snippets can break the main server, so `nginx -t` must gate every write/delete.

**Reversal cost.** Medium; changing to host-based routing would alter all env URLs.

**Citations.**
- `governance/10_API_CONTRACT.md` State Schema
- `governance/06_DEFINITION_OF_DONE.md` `nginx/` section

## LEARNINGS

- BusyBox `mktemp` requires the `XXXXXX` token at the end of the template; the atomic writer now uses a portable temp-file pattern.
- Destroy is faster and cleaner when the app container is removed before verifying the `docker logs -f` PID; the log shipper exits naturally instead of waiting for the full SIGTERM timeout.
