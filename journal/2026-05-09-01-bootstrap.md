## TEACH-BACKS

### TEACH-BACK: WSL2 Ubuntu-22.04 over Windows-native pre-commit run

**Context.** Sprint 0's exit gate requires `pre-commit run --all-files` exits 0 with zero skipped hooks. The hook set includes `shellcheck`, `shfmt`, `hadolint`, `gitleaks`, and `ruff`. The host is Windows 11; the deployment target is a Linux VM (per the brief, "everything must run on a single Linux VM"). Running pre-commit on Windows directly forces several hooks into "skipped on this platform" or "binary not found" failure modes, which violates the gate.

**Alternatives considered**
1. **Windows-native pre-commit, accept skips.** Fastest start, but `shellcheck-precommit` and `shfmt` rely on POSIX shell heredoc semantics that misbehave on Windows; `hadolint-docker` requires Docker which exists here only via Docker Desktop's WSL2 backend anyway. Skips would compound and the gate would be a lie.
2. **WSL2 Ubuntu-22.04 (chosen).** Linux parity with the deployment target; every hook runs natively; matches the CI runner (`ubuntu-22.04`). Costs ≈ 30 min of one-time setup and a reboot.
3. **Docker-only execution via `docker run` per hook.** Avoids WSL but requires hand-rolling per-hook container invocations, which is exactly what `pre-commit` already abstracts. Throws away the framework's value.
4. **CI-only enforcement, skip local run.** Pre-commit is meant to be a local gate; deferring to CI means broken commits get pushed and the loop's verify step (`make ship-check`) becomes meaningless on the dev machine.

**Chosen** — option 2. WSL2 makes the dev environment a faithful subset of the production environment, satisfying the spirit of the "single Linux VM" constraint while still letting Kelechi keep his Windows host. The reboot cost is one-time.

**Failure modes**
- WSL provisioning fails (kernel update missing, virtualization disabled in BIOS). Mitigation: hard 45-minute ceiling per Kelechi's instruction. If blown, fall back to Windows-native pre-commit with a `docs/divergence_log.md` entry documenting which hooks are skipped and why.
- Git line endings drift between Windows checkouts and WSL checkouts. Mitigation: `.gitattributes` enforces `* text=auto eol=lf` and `*.sh eol=lf`, and `mixed-line-ending --fix=lf` is in pre-commit.
- Filesystem performance over `/mnt/c/...` is poor enough that `pre-commit run --all-files` becomes painful. Mitigation: clone the repo at `~/devops-sandbox` inside the WSL filesystem, not via the Windows mount.

**Reversal cost.** Trivial — uninstalling Ubuntu is one command (`wsl --unregister Ubuntu-22.04`). The repo files are platform-agnostic.

**Citations.**
- Microsoft, "WSL2 file system performance vs `/mnt/c`" — the 9P file-share overhead is documented and real.
- `governance/07_DEVSECOPS_GUARDRAILS.md` §8 (the full pre-commit hook list).
- Kelechi's Sprint 0 instruction (this conversation) — explicit hard ceiling and divergence-log fallback.

### TEACH-BACK: Local pre-commit hooks as real scripts, gated only on `feat:`/`fix:` for the journal hook

**Context.** `.pre-commit-config.yaml` declares two local hooks: `governance-matrix-touched` and `journal-entry-on-feat`. The shipped pack references `scripts/check_acceptance_matrix.sh` and `scripts/check_journal_today.sh`, but neither file exists. With pre-commit's local-hooks contract, the absence of the script files would make the hooks fail every commit. We need real implementations.

**Alternatives considered**
1. **Stub them with `exit 0`.** Hooks load, but the traceability gate AGENTS.md §3 step 6 depends on (matrix updated when functional code changes) becomes a no-op until Sprint 1. Sprint 0 would tick C02 ("pre-commit hooks installed") on a lie.
2. **Disable the `repo: local` block.** Cleanest pre-commit run but contradicts the governance pack's intent and forfeits the gate.
3. **Real implementations now (chosen).** A few extra lines of bash today buys us the gate from day one. The complexity is low: matrix hook greps `git diff --cached --name-only` for files under `platform|monitor|nginx|Makefile` and asserts `governance/03_ACCEPTANCE_CRITERIA.md` is also staged; journal hook reads `$1` (the commit message file path on commit-msg stage), extracts the type prefix, and only enforces "today's journal exists" for `feat:` and `fix:`.

**Chosen** — option 3. Both scripts use `set -euo pipefail`, fail with a message that points at `governance/03_ACCEPTANCE_CRITERIA.md` or `governance/05_JOURNAL_PROTOCOL.md` so future-me knows where to look, and respect the standard `SKIP=hook-id git commit ...` escape hatch by emitting a "this skip is logged" warning to stderr (pre-commit's framework already supports `SKIP`; we just print a notice).

**Why exempt `docs:`/`chore:`/`ci:`/`refactor:`/`test:`/`build:`/`perf:` from the journal hook.** The journal protocol says "one entry per working session." A documentation cleanup commit, a CI tweak, or a chore commit (e.g., bumping a pinned version) is not a "working session" — it is a small administrative change that does not advance an acceptance row and therefore does not need a teach-back block or a fresh journal entry. Forcing a journal entry for those would either pollute the journal with empty stub entries (the LEARNINGS section of a commit-message-typo fix is the empty string) or train me to mis-classify substantive changes as `chore:` to dodge the hook. The journal hook fires on `feat:` and `fix:` because those are the commit types that close acceptance rows; the matrix hook independently catches the case where someone tries to ship functional change disguised as a `chore:` (it triggers on file paths, not on commit type, so a `chore:` commit that touches `platform/` still gets blocked).

**Failure modes**
- A commit that legitimately needs to touch `platform/` for a tooling-only reason (e.g., `chmod +x` an existing script) gets blocked. Mitigation: documented `SKIP=governance-matrix-touched` escape hatch.
- The journal hook misreads the commit message because it lives at `$1` only in the `commit-msg` stage; if pre-commit's stage detection misfires, the script may try to read a non-existent path. Mitigation: `[[ -r "${1:-}" ]] || exit 0` early-return — if we cannot read the commit message file, we silently pass rather than block.
- I forget to update `governance/03_ACCEPTANCE_CRITERIA.md` C06 row when these scripts evolve. Mitigation: the matrix-touched hook itself triggers on changes to `scripts/check_*.sh`.

**Reversal cost.** Trivial. Both scripts are <50 lines each.

**Citations.**
- pre-commit docs, "Local hooks" section.
- `governance/03_ACCEPTANCE_CRITERIA.md` (the document the hook protects).
- `governance/05_JOURNAL_PROTOCOL.md` "Linking commits to entries" — establishes the `feat:`/`fix:` invariant.

### TEACH-BACK: Copy `governance/ci/*` into `ci/` rather than symlink

**Context.** `.pre-commit-config.yaml` references `ci/gitleaks.toml`. `.github/workflows/ci.yml` references `ci/shellcheck.sh` and `ci/gitleaks.toml`. The source-of-truth files live under `governance/ci/`. Sprint 0's task list says "Symlink (or copy) `governance/ci/pre-commit-config.yaml` → `.pre-commit-config.yaml` and same for gitleaks."

**Alternatives considered**
1. **Symlink `ci/ -> governance/ci/`.** One file, zero duplication. Symlinks behave inconsistently across Windows/WSL/Linux checkouts; `git` on Windows often materialises symlinks as plaintext files containing the target path, which would break the hook config in subtle ways. Also: symlinks across the governance/ci → ci boundary couple "ground truth" to "operational config", and an agent doing Sprint 5 polish on `ci/gitleaks.toml` could accidentally mutate the governance copy.
2. **Copy with a `# GENERATED — do not edit by hand` header pointing at the source (chosen).** Two-line cost, deterministic across platforms, and the header signals to future-me that the source-of-truth lives elsewhere. Constitution §1.3 requires this header on generated files anyway.
3. **Move the files outright (no copy in `governance/ci/`).** Cleanest in one sense, but the governance pack is meant to be self-contained — a future migration to a different directory layout should not require chasing references back into `ci/`.

**Chosen** — option 2. Symlinks are not portable enough to bet a Sprint 0 gate on. The duplication is one diff away from being detected (a Sprint 5 hygiene pass can grep for "GENERATED" headers and confirm contents match `governance/ci/`).

**Failure modes**
- The two copies drift. Mitigation: pre-commit hook in a future sprint that diffs `ci/*.{toml,sh}` against `governance/ci/*.{toml,sh}` and refuses divergence. (Not Sprint 0 work; logged as future improvement in NOTES.)
- The header gets stripped by an over-eager `end-of-file-fixer` or `trailing-whitespace` pass. Mitigation: the header is a normal comment line in TOML/bash; nothing in the configured hook set touches it.

**Reversal cost.** One-line `cp` to re-sync; trivial.

**Citations.** Constitution §1.3.

## LEARNINGS

- **TOML literal strings (`'''...'''`) do not process escape sequences.** I assumed `\\` would be a single backslash like in YAML or JSON. It is two literal backslashes. So `'''(.*\\.example$)'''` compiles to the regex `.*\\.example$`, which requires a literal backslash before `.example` and silently never matches `.env.example`. Lesson: in TOML literal strings, regex metacharacters like `\.` and `\s` go in directly, no doubling. The bug had been latent in the governance pack since I authored it; my own .env.example tripped it. The whole gitleaks-can't-find-the-secret-it-just-flagged class of false positives often traces back to mis-escaped path allowlists.
- **`\s*` in regex matches newlines, which makes "this looks like one-line config" rules fire across blank lines.** `API_TOKEN=\n\nDEFAULT_TTL_MIN=30` made `DEFAULT_TTL_MIN` look like the "secret value of API_TOKEN" because `\s*` swallowed the two `\n`s. Use `[ \t]*` when you specifically want horizontal whitespace.
- **`readonly var="$(cmd)"` masks the return code of `cmd`.** ShellCheck SC2155. The `readonly` builtin always succeeds, even if the command-substitution failed and assigned the empty string. Split: `var="$(cmd)"; readonly var`. Easy to miss in fast-written helper scripts.
- **PowerShell expands `$VAR` references inside what *looks* like a single-quoted bash payload when invoking `wsl -d Ubuntu -- bash -c '...'`.** I lost ~15 minutes thinking rsync had wandered into Visual Studio Build Tools, when in fact `$SRC`/`$DST` had been silently consumed by PowerShell's variable expansion before the string ever reached WSL bash, and rsync defaulted to walking from CWD. Workaround: write the bash payload to a file (Write tool, no interpolation), `cp` it into the WSL repo, execute via `bash <absolute-path>`. I am now using this pattern for every multi-line bash invocation that crosses the bridge.
- **`hashFiles()` at GitHub Actions JOB level can break the workflow's parse, not just skip the job.** I tried `if: hashFiles('docker-compose.yml') != ''` at the build-test and trivy job-level conditions, expecting GitHub to skip them in Sprint 0 and run them in Sprint 1+. Result: GitHub failed to schedule ANY jobs (`jobs: []`), reported "This run likely failed because of a workflow file issue", and even renamed the workflow from `ci` to its file path. The function is reliable only inside a step that runs after `actions/checkout` (i.e., when the workspace exists). The fix that landed: keep `if: hashFiles('**/Dockerfile') != ''` on the hadolint step (well after checkout, well-supported), and stop trying to gate jobs at the job level. build-test and trivy stay red in Sprint 0 — that's expected and acceptable; they go green in Sprint 1 when their artefacts land.
- **`gitleaks/gitleaks-action@v2` cannot scan a brand-new repo on the very first push.** It computes the diff range as `<before>^..<after>`. On the first push there is no parent of the first commit, so git errors and the action exits 1 with `scanned ~0 bytes`. The fix is to invoke the gitleaks binary directly (`curl | tar | ./gitleaks detect --no-git --redact -v --source .`) — same scan as we run locally, no git-history dance. For history scanning we will add a separate scheduled workflow in a later sprint, when the repo has more than one commit's worth of history to look at.
