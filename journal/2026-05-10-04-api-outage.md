## TEACH-BACKS

### TEACH-BACK: FastAPI wraps scripts instead of reimplementing Docker logic

**Context.** The brief says the control API wraps the scripts. Reusing the lifecycle scripts keeps Docker naming, labels, Nginx reload behavior, and atomic state writes in one implementation.

**Alternatives considered**
1. **API calls Docker directly** — easier response handling, but duplicates create/destroy logic and risks drift.
2. **API shells out to scripts and reads state files** — one source for lifecycle behavior, with Python only shaping HTTP responses.

**Chosen** — **script wrapping**, because it satisfies A36 and keeps the Constitution-sensitive behavior in the bash helpers.

**Failure modes**
- Script stdout becomes an API contract dependency; the API should primarily read state files after script success.
- Long Docker operations can hang requests; subprocess calls need strict timeouts.

**Reversal cost.** Medium. Direct Docker API use would touch every endpoint and test.

**Citations.**
- `governance/CURRENT_TASK` §7
- `governance/06_DEFINITION_OF_DONE.md` `api.py`

### TEACH-BACK: Bash guard plus API-side policy-shaped precheck

**Context.** CRISIS mode says outage simulation must guard in bash and check the existing OPA policy from the API. The repo has Rego policy files but no OPA binary in the approved stack.

**Alternatives considered**
1. **Shell out to `opa eval`** — exact policy execution, but adds a dependency not listed in the stack and triggers an ask/stop condition.
2. **Mirror the Rego input constraints in API code and keep the bash `assert_app_container` guard as the enforcement backstop** — no new dependency, same protected-container/mode/state rules.

**Chosen** — **policy-shaped precheck without adding OPA**, because adding a binary dependency violates the crisis "move" constraint and the bash guard remains authoritative.

**Failure modes**
- The Python mirror can drift from `policies/sandbox_outage.rego`; keep the input fields and rule names aligned in comments.
- A direct CLI call can bypass API precheck, so `simulate_outage.sh` must still call `assert_app_container` first.

**Reversal cost.** Low. Installing OPA later only changes one API helper.

**Citations.**
- `CRISIS_PROMPT.md` non-negotiable outage guard
- `policies/sandbox_outage.rego`

## LEARNINGS

- The shipped Postman pack is stricter than the prose contract in a few places: create responses need `expires_at`, outage responses use `env_id`, and validation failures are asserted as 422.
- FastAPI treats `Response | JSONResponse` return annotations as response models unless disabled; route handlers that return mixed response classes should omit that annotation.
